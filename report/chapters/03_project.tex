\chapter{Phân tích và Thiết kế hệ thống Trello Mini}

\section{Tổng quan Đồ án (Capstone Project)}
Phần đồ án tổng hợp này là nơi lắp ráp toàn bộ các module kỹ thuật, kiến trúc và công cụ đã trình bày ở phần trước (React Router, Zustand, FSD, Custom Hooks, Tailwind CSS) thành một hệ thống Kanban hoàn chỉnh và thống nhất. Trello Mini không chỉ là một trang web tĩnh, nó là một Web App tương tác cao.

Các chức năng chính đã được triển khai thành công bao gồm:
\begin{enumerate}
    \item \textbf{Quản lý Authentication:} Đăng nhập mô phỏng và phân quyền định tuyến.
    \item \textbf{Dashboard (Bảng điều khiển):} Quản lý nhiều dự án/bảng khác nhau (Tạo mới, đổi tên, xóa bảng).
    \item \textbf{Giao diện Kanban (Board view):} Quản lý các danh sách (Todo, In Progress, Done) và thẻ công việc (Cards).
    \item \textbf{Drag and Drop (Kéo và Thả):} Cho phép kéo thẻ giữa các danh sách hoặc trong cùng danh sách, cập nhật thứ tự tự động.
\end{enumerate}

\chapter{Quản lý Bảng dự án (Dashboard)}

\section{Cơ chế Phản hồi (Optimistic UI)}
Trong dự án này, chúng tôi áp dụng tư duy "Optimistic UI" (Giao diện lạc quan). Ngay khi form được gửi (Submit), bảng mới lập tức xuất hiện trên màn hình nhờ vào khả năng re-render siêu tốc của React khi State thay đổi.

\chapter{Triển khai Bảng Kanban và Kéo Thả (Drag and Drop)}

\section{Cấu trúc Component lồng nhau (Nested Components)}
Màn hình chi tiết bảng (\texttt{BoardPage}) là phần lõi phức tạp nhất của ứng dụng. Kiến trúc của nó thể hiện rõ ràng sức mạnh của mô hình Component-based trong React:
\begin{itemize}
    \item Component \texttt{BoardPage}: Chứa tiêu đề bảng, gọi và lặp qua các mảng danh sách (Lists).
    \item Component \texttt{List}: Hiển thị tiêu đề của cột (Todo, Doing), nút "Thêm thẻ" và lặp qua mảng các thẻ.
    \item Component \texttt{TaskCard}: Hiển thị nội dung chi tiết của một công việc.
\end{itemize}

\section{Áp dụng Kéo Thả (Drag and Drop) vào Bảng Kanban}

\subsection{Luồng hoạt động khi Kéo thẻ giữa các Danh sách}
Việc kéo thả thẻ (Card) từ danh sách này sang danh sách khác hoặc thay đổi vị trí trong cùng danh sách đòi hỏi hệ thống State phải vô cùng mạnh mẽ để tính toán lại mảng (Array Reordering).

Khi người dùng nhấn giữ và kéo thẻ (hành động được bắt bởi \texttt{useSensors}), thư viện sẽ tính toán khoảng cách và vị trí va chạm, sau đó phát ra sự kiện \texttt{onDragEnd}.

\begin{Verbatim}
// Trích từ: src/pages/boards/BoardPage.tsx
const handleDragEnd = (event: DragEndEvent) => {
  const { active, over } = event;

  if (!over || active.id === over.id) return;

  const activeData = active.data.current;
  const overData = over.data.current;

  if (activeData?.type === "Card") {
    const activeCardId = active.id;
    const fromListId = activeData.listId;

    // Thả lên Card khác -> lấy listId của Card đó; thả lên List -> dùng over.id
    const toListId = overData?.type === "Card" ? overData.listId : over.id;

    // Vị trí đích lấy từ sortable index do dnd-kit tính toán
    const overIndex = overData?.sortable?.index;

    if (boardId && fromListId && toListId) {
      moveCard(boardId, fromListId, toListId, activeCardId, overIndex);
    }
  }
};
\end{Verbatim}

Tại đây, action \texttt{moveCard} (đã xây dựng trong Zustand với Immer) được gọi cho cả hai trường hợp: di chuyển thẻ trong cùng danh sách và chuyển sang danh sách khác. Tham số \texttt{overIndex} được lấy từ \texttt{over.data.current?.sortable?.index} do \texttt{@dnd-kit/sortable} tự động gắn vào mỗi phần tử có thể sắp xếp.

\chapter{Kết luận việc triển khai Đồ án}
Tất cả các kiến thức từ quá trình khởi tạo dự án với Vite, thiết kế hệ thống giao diện với Tailwind CSS, quản lý State phức tạp với Zustand, phân quyền với React Router, và xử lý logic với Custom Hooks đều đã được chứng minh hiệu quả thực tế thông qua Trello Mini. Giao diện mượt mà, tốc độ phản hồi tức thì (Optimistic UI) và mã nguồn dễ bảo trì chính là tính ưu việt mà React mang lại.
