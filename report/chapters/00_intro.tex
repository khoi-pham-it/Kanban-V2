\section{Lý do chọn đề tài}
Mô hình Kanban (Todo $\rightarrow$ Doing $\rightarrow$ Done) đã chứng minh được hiệu quả vượt trội trong việc tối ưu hóa quy trình làm việc.

Nhóm chọn đề tài xây dựng ứng dụng \textbf{Trello Mini} sử dụng \textbf{ReactJS} vì:
\begin{itemize}
    \item \textbf{Kiến trúc dựa trên Component:} Rất phù hợp để xây dựng các thành phần lồng nhau của bảng Kanban (Board > List > Card).
    \item \textbf{State-driven rendering:} Giúp cập nhật giao diện ngay lập tức khi trạng thái công việc thay đổi (kéo thả card).
    \item \textbf{Hệ sinh thái phong phú:} Các thư viện như \texttt{dnd-kit}, \texttt{React Router} giúp xử lý các bài toán phức tạp một cách hiệu quả.
\end{itemize}

\section{Mục tiêu báo cáo}
\begin{itemize}
    \item \textbf{Mục tiêu về kiến thức:} Hiểu sâu về React Hooks (useAuth, useBoards, useCards), Context API, và quy trình xử lý State trong ứng dụng phức tạp.
    \item \textbf{Mục tiêu về kỹ năng:} Xây dựng hoàn chỉnh một ứng dụng quản lý công việc có tính năng kéo thả, phân quyền truy cập (Protected Routes) và lưu trữ dữ liệu bền vững (Local Storage).
\end{itemize}

\section{Phạm vi nghiên cứu}
\begin{itemize}
    \item Tập trung vào xử lý logic phía Client-side.
    \item Mô phỏng quá trình Authentication (Fake Auth).
    \item Quản lý dữ liệu tập trung qua Context API thay vì dùng Backend thực tế trong giai đoạn này.
\end{itemize}
