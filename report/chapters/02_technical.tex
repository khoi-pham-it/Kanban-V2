\end{itemize}

\section{Kiến trúc Siêu tốc của Vite: Deep Dive}
Thay vì sử dụng các công cụ đóng gói truyền thống như Webpack vốn đã trở nên già cỗi và chậm chạp, dự án lựa chọn \textbf{Vite}. Để hiểu tại sao Vite lại là "kẻ thay đổi cuộc chơi", ta cần đi sâu vào kiến trúc bên trong của nó.

\subsection{ESBuild vs Rollup: Sự kết hợp hoàn hảo}
Vite sử dụng hai trình đóng gói khác nhau cho hai mục đích khác nhau, tối ưu hóa cả quá trình phát triển lẫn sản phẩm đầu ra:
\begin{itemize}
    \item \textbf{Development (ESBuild):} Trong quá trình phát triển, Vite sử dụng \texttt{esbuild} để pre-bundle các thư viện phụ thuộc (dependencies). \texttt{esbuild} được viết bằng Go và có tốc độ nhanh hơn từ 10 đến 100 lần so với các công cụ dựa trên JavaScript. Nó tận dụng tối đa đa luồng (multi-threading) của CPU, giúp server khởi động gần như tức thì ngay cả với hàng trăm dependency.
    \item \textbf{Production (Rollup):} Đối với bản build cuối cùng, Vite tin dùng \texttt{Rollup}. Mặc dù chậm hơn ESBuild, Rollup cung cấp khả năng tối ưu hóa mã nguồn cực cao như:
    \begin{itemize}
        \item \textit{Tree-shaking:} Loại bỏ các đoạn code không bao giờ được dùng đến (dead code).
        \item \textit{Scope Hoisting:} Gộp các module lại để giảm kích thước file và tăng tốc độ thực thi.
        \item \textit{Advanced Chunking:} Chia nhỏ file thông minh để tận dụng cơ chế cache của trình duyệt.
    \end{itemize}
\end{itemize}

\subsection{Native ESM: Giải mã sự khác biệt với Webpack}
Webpack hoạt động theo cơ chế "Bundle-based": Nó phải quét toàn bộ ứng dụng, xây dựng cây phụ thuộc (Dependency Graph) và đóng gói tất cả thành một file khổng lồ (bundle) trước khi có thể hiển thị trang web. Khi dự án lớn lên, thời gian đợi chờ này tăng theo cấp số nhân ($O(n)$).

Vite giải quyết bài toán này bằng cách sử dụng \textbf{Native ESM (ES Modules)} của trình duyệt. Thay vì đóng gói, Vite chỉ đơn giản là phục vụ các file nguồn theo yêu cầu của trình duyệt. Nhờ đó, tốc độ khởi động của Vite là hằng số ($O(1)$), không phụ thuộc vào độ lớn của dự án.

\chapter{Hệ thống Routing và Lazy Loading Component}

\section{Routing thủ công với React Router}
Trong các Single Page Application, việc điều hướng giữa các trang diễn ra ở phía client mà không cần tải lại toàn bộ tài liệu HTML. React Router là thư viện tiêu chuẩn để quản lý việc này. Trong dự án, chúng tôi tự cấu hình React Router kết hợp với việc phân quyền (Protected Routes) để đảm bảo an ninh.

\subsection{Bảo vệ Tuyến đường (Protected Routes)}
Bảo vệ tuyến đường là kỹ thuật ngăn chặn người dùng chưa đăng nhập truy cập vào các trang nhạy cảm (như Dashboard hay bảng công việc).

\begin{Verbatim}
// Code mẫu: src/components/guard/ProtectedRoute.tsx
import { Navigate, Outlet } from "react-router-dom";
import { useAuthStore } from "../../features/auth/store/useAuthStore";

const ProtectedRoute = () => {
  const isAuthenticated = useAuthStore((state) => state.isAuthenticated);

  if (!isAuthenticated) {
    return <Navigate to="/login" replace />;
  }

  return <Outlet />;
};

export default ProtectedRoute;
\end{Verbatim}

\section{Lazy Loading Component}
Để tối ưu hóa thời gian tải trang ban đầu (First Contentful Paint), ứng dụng sử dụng kỹ thuật Lazy Loading. Thay vì gói toàn bộ mã nguồn vào một file JavaScript khổng lồ, React chia nhỏ mã nguồn và chỉ tải về (fetch) các component khi người dùng thực sự điều hướng đến trang đó.

\subsection{Sử dụng React.lazy và Suspense}
Kỹ thuật này được thực hiện thông qua hàm \texttt{lazy()} của React, kết hợp với component \texttt{<Suspense>} để hiển thị giao diện chờ (fallback) trong thời gian tải.

\textbf{Bài tập nhỏ 1 \& 2: Thiết lập Routing thủ công và Lazy Loading}
\begin{enumerate}
    \item \textbf{Bước 1:} Sử dụng \texttt{lazy()} để import động các trang.
    \item \textbf{Bước 2:} Tạo \texttt{SuspenseWrapper} để hiển thị trạng thái đang tải.
    \item \textbf{Bước 3:} Khai báo danh sách các route bằng \texttt{createBrowserRouter}, áp dụng \texttt{ProtectedRoute} cho nhóm route cần bảo vệ.
\end{enumerate}

\begin{Verbatim}
// Code mẫu: Cấu hình Router (src/app/router.tsx)
import { lazy, Suspense } from "react";
import { createBrowserRouter, redirect } from "react-router-dom";
import ProtectedRoute from "../components/guard/ProtectedRoute";

const Login = lazy(() => import("../pages/auth/LoginPage"));
const Dashboard = lazy(() => import("../pages/boards/DashboardPage"));
const BoardDetail = lazy(() => import("../pages/boards/BoardPage"));

const SuspenseWrapper = ({ children }: { children: React.ReactNode }) => (
  <Suspense fallback={<div>Đang tải giao diện...</div>}>
    {children}
  </Suspense>
);

export const router = createBrowserRouter([
  { path: "/", loader: () => redirect("/boards") },
  { path: "/login", element: <SuspenseWrapper><Login /></SuspenseWrapper> },
  {
    path: "/boards",
    element: <ProtectedRoute />, // Bảo vệ toàn bộ route /boards
    children: [
      { index: true, element: <SuspenseWrapper><Dashboard /></SuspenseWrapper> },
      { path: ":boardId", element: <SuspenseWrapper><BoardDetail /></SuspenseWrapper> },
    ],
  },
]);
\end{Verbatim}

\chapter{Quản lý State Toàn cục và Custom Hooks}

\section{Quản lý State phức tạp với Zustand}
Zustand được chọn làm giải pháp quản lý State toàn cục thay vì Redux hay Context API. Với các ứng dụng phức tạp như Kanban, việc chỉ sử dụng \texttt{useState} đơn giản là không khả thi do hiện tượng "Prop Drilling" (phải truyền state qua quá nhiều component trung gian). Zustand giải quyết vấn đề này với cú pháp cực kỳ ngắn gọn và hiệu năng cao.

\subsection{Tích hợp Middleware: Persist và Immer}
Việc quản lý các cấu trúc dữ liệu lồng nhau (Nested data) rất khó khăn do nguyên tắc bất biến (Immutability) của React. Zustand cho phép sử dụng \texttt{Immer} để viết code cập nhật state như thể đang "đột biến" (mutate) dữ liệu, nhưng thực chất Immer sẽ tạo ra bản sao bất biến ngầm bên dưới.

Đồng thời, middleware \texttt{persist} được sử dụng để tự động lưu trạng thái vào \texttt{LocalStorage}, giúp dữ liệu không bị mất khi làm mới trang.

\textbf{Bài tập nhỏ 3: Xây dựng Store quản lý Board với Zustand}
\begin{enumerate}
    \item \textbf{Bước 1:} Tạo một slice quản lý Board (\texttt{boardSlice.ts}) bằng \texttt{StateCreator} của Zustand.
    \item \textbf{Bước 2:} Khởi tạo các hàm actions (như \texttt{addBoard}, \texttt{updateBoard}). Nhờ có Immer, việc thêm phần tử vào mảng chỉ cần \texttt{state.boards.push(...)} cực kỳ tự nhiên.
    \item \textbf{Bước 3:} Gom các slice lại thành store chính và bọc bởi \texttt{persist} middleware.
\end{enumerate}

\begin{Verbatim}
// Code mẫu: Tạo Slice cho Board (src/features/kanban/store/slices/boardSlice.ts)
export const createBoardSlice: StateCreator<
  StoreState,
  [["zustand/immer", never]],
  [],
  BoardSlice
> = (set) => ({
  boards: initialBoards,
  addBoard: (title) =>
    set((state) => {
      state.boards.push({
        id: `board-${Date.now()}`,
        title,
        lists: [],
      });
    }),
});

// Code mẫu: Tạo Global Store (src/features/kanban/store/useBoardStore.ts)
import { create } from "zustand";
import { persist } from "zustand/middleware";
import { immer } from "zustand/middleware/immer";

export const useBoardStore = create<StoreState>()(
  persist(
    immer((...a) => ({
      ...createBoardSlice(...a),
      // ...các slice khác
    })),
    { name: "trello-storage" },
  )
);
\end{Verbatim}

\section{Custom Hooks: Tách biệt Logic khỏi Giao diện}
Custom Hooks là công cụ mạnh mẽ nhất của React để tái sử dụng logic (Stateful logic) giữa các component khác nhau. Việc đưa logic vào Custom Hook giúp Component giao diện trở nên mỏng nhẹ và dễ đọc hơn.

\textbf{Bài tập nhỏ 4 \& 5: Tự viết Custom Hooks}
Trong dự án, chúng tôi xây dựng hai Custom Hooks thiết yếu: \texttt{useClickOutside} và \texttt{useForm}.

\subsection{Hook useClickOutside}
Hook này giúp phát hiện sự kiện click chuột ra bên ngoài một khu vực nhất định, rất hữu ích cho các UI component như Dropdown, Modal hoặc Inline-edit Input.

\begin{Verbatim}
// Code mẫu: src/hooks/useClickOutside.ts
import { useEffect, type RefObject } from 'react';

export function useClickOutside<T extends HTMLElement>(
  ref: RefObject<T | null>,
  handler: (event: MouseEvent | TouchEvent) => void
) {
  useEffect(() => {
    const listener = (event: MouseEvent | TouchEvent) => {
      const el = ref?.current;
      // Nếu click vào chính element hoặc con của nó thì bỏ qua
      if (!el || el.contains((event?.target as Node) || null)) return;
      handler(event);
    };

    document.addEventListener('mousedown', listener);
    return () => document.removeEventListener('mousedown', listener);
  }, [ref, handler]);
}
\end{Verbatim}

\subsection{Hook useForm}
Hook quản lý trạng thái của một form bất kỳ, tự động xử lý các sự kiện \texttt{onChange}, xác thực lỗi (validate) và \texttt{onSubmit}.

\begin{Verbatim}
// Code mẫu: src/hooks/useForm.ts
import { useState, type ChangeEvent } from 'react';

export function useForm<T>({ initialValues, onSubmit, validate }: UseFormProps<T>) {
  const [values, setValues] = useState<T>(initialValues);
  const [errors, setErrors] = useState<Partial<Record<keyof T, string>>>({});

  const handleChange = (e: ChangeEvent<HTMLInputElement>) => {
    const { name, value } = e.target;
    setValues({ ...values, [name]: value });
    if (errors[name as keyof T]) {
      setErrors({ ...errors, [name]: undefined });
    }
  };

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();
    if (validate) {
      const validationErrors = validate(values);
      if (Object.keys(validationErrors).length > 0) {
        setErrors(validationErrors);
        return;
      }
    }
    onSubmit(values);
  };

  return { values, errors, handleChange, handleSubmit };
}
\end{Verbatim}

\chapter{Tương tác UI Nâng cao: Kéo Thả (Drag and Drop)}

\section{Tổng quan về thư viện dnd-kit}
Việc xây dựng một hệ thống kéo thả (Drag and Drop - DnD) từ đầu bằng các sự kiện chuột (Mouse events) của DOM là cực kỳ phức tạp và dễ phát sinh lỗi, đặc biệt là trên các thiết bị cảm ứng (Touch devices) và trình đọc màn hình (Screen readers). Để giải quyết bài toán này trong hệ sinh thái React, \textbf{dnd-kit} là thư viện hiện đại và tối ưu nhất tính tới thời điểm hiện tại.

Khác với các thư viện cũ như \texttt{react-beautiful-dnd}, \texttt{dnd-kit} được thiết kế theo kiến trúc module hóa:
\begin{itemize}
    \item \textbf{@dnd-kit/core}: Chứa logic cốt lõi (Core logic), quản lý ngữ cảnh (Context) và các cảm biến (Sensors).
    \item \textbf{@dnd-kit/sortable}: Chuyên biệt cho bài toán sắp xếp mảng (Sortable lists).
    \item \textbf{@dnd-kit/utilities}: Cung cấp các hàm hỗ trợ (như \texttt{arrayMove} để hoán đổi vị trí phần tử).
\end{itemize}

\section{Kiến trúc hoạt động của dnd-kit}

\subsection{DndContext và Sensors}
Thành phần gốc của thư viện là \texttt{<DndContext>}. Nó bao bọc khu vực giao diện mà chúng ta muốn có khả năng kéo thả, quản lý trạng thái kéo và phát ra các sự kiện quan trọng như \texttt{onDragStart}, \texttt{onDragOver}, và \texttt{onDragEnd}.

\textbf{Sensors} (Cảm biến) quyết định khi nào thì một hành động kéo (Drag) bắt đầu. Trong dự án Kanban, chúng ta kết hợp cả \texttt{PointerSensor} (Dành cho chuột trên Desktop) và \texttt{TouchSensor} (Dành cho chạm trên Mobile) để ứng dụng hoạt động đa nền tảng.

\subsection{SortableContext và useSortable Hook}
Bên trong \texttt{DndContext}, chúng ta sử dụng \texttt{<SortableContext>} để báo cho thư viện biết danh sách các mục (items) nào có thể sắp xếp.

Hook \texttt{useSortable} được áp dụng lên từng phần tử con (ví dụ: từng Thẻ công việc) để gắn các thuộc tính (attributes) và lắng nghe (listeners) vào thẻ HTML, giúp nó có thể "bị nắm lấy" và "di chuyển". Hook này trả về \texttt{transform} (tọa độ x, y khi đang kéo) và \texttt{transition} (hiệu ứng chuyển động mượt mà).

\textbf{Bài tập nhỏ 6: Áp dụng dnd-kit cho một Component có thể kéo (Draggable)}
\begin{enumerate}
    \item \textbf{Bước 1:} Sử dụng hook \texttt{useSortable} truyền vào \texttt{id} và \texttt{data} (thông tin đính kèm của phần tử).
    \item \textbf{Bước 2:} Lấy ra \texttt{attributes}, \texttt{listeners}, \texttt{setNodeRef}, \texttt{transform}, \texttt{transition}.
    \item \textbf{Bước 3:} Áp dụng CSS transform dựa trên tọa độ \texttt{transform} trả về để đối tượng di chuyển theo con trỏ chuột.
\end{enumerate}

\begin{Verbatim}
// Code mẫu: Thiết lập phần tử có thể kéo (src/features/kanban/components/TaskCard.tsx)
import { useSortable } from "@dnd-kit/sortable";
import { CSS } from "@dnd-kit/utilities";

export const TaskCard = ({ card, listId }: TaskCardProps) => {
  const { setNodeRef, attributes, listeners, transform, transition, isDragging } =
    useSortable({
      id: card.id,
      data: { type: "CARD", card, listId },
    });

  const style = {
    transition,
    transform: CSS.Transform.toString(transform),
  };

  if (isDragging) {
    return (
      <div ref={setNodeRef} style={style} className="opacity-50 border-2 border-blue-500 bg-blue-50">
        Đang di chuyển...
      </div>
    );
  }

  return (
    <div ref={setNodeRef} style={style} {...attributes} {...listeners} className="bg-white p-3 rounded shadow-sm cursor-grab">
      {card.title}
    </div>
  );
};
\end{Verbatim}
